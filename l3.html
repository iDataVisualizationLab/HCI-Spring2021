Principles

Recognize Diversity

  • Diversity of users
    User profile: Age, gender, preferred language, education, cultural background, job, and technical skill.

    User classes:
    • novice / first time users
      • novice - know little of task or interface concepts
      • first time users - have task concepts but shallow knowledge of interface

    • knowledgeable intermittent users
      • know about the task and general knowledge about the interface - will forget specific features
      • need to easily rediscover how to perform tasks

    • expert frequent users
      • thoroughly familiar with task and interface concepts
      • need ability to quickly carry out actions, get rapid responses, brief feedback
         
  • Diversity of tasks
    • determine set of tasks early in design
    • choose 'atomic' actions for the tasks
    • look at task frequencies --> prioritize tasks
       
      •           
  • Diversity of interaction styles

Direct manipulation - originally interacting with a computer was direct manipulation at a low level - moving wires, flipping switches. Direct manipulation returned in a big way in the 80s (after a lot of basic research was done in the 60s and 70s) at a higher user level which could involve using a light pen to touch a location on the screen (70s), moving a mouse to move a pointer on a screen and clicking the mouse button to make selections (80s) to more modern alternatives where the user uses a stylus or their finger to more directly manipulate the interface itself (2000s).
Definition: A human-computer interaction style which involves continuous representation of objects of interest, and rapid, reversible, incremental actions and feedback

Golden rules of interface design

  • Strive for consistency
    1. The content, the UI elements (such as menus), terminology, commands, fonts, backgrounds, and colors should be in harmony
    2. The layouts of various pages/sections in your application

      3. Images are from www.interaction-design.org
    3. Consistency across apps in the same system: iPhone app layout and user gestures.
    4. Consistency across devices: TVs, PCs, Tablets, and smartphones.

    Benefits of Consistency in UI Design. Examples
    1. Reduce Learning and increase Usability: Consistency makes it easier for your users to navigate and use your app because they don’t have to learn new ropes to get around.
    2. Eliminates Confusion: By using consistency to create a logical structure and to clearly define where users can find what
    3. Evoke an Emotional Response: Consistency helps your users feel confident in their ability to manage what’s under their thumbs and will evoke a pleasant experience.
  • Provide universal usability
    • help new users through basic procedures
    • enable frequent users to use shortcuts. Expert users might find the following features helpful: Abbreviations, Function keys, Hidden commands, and Macro facilities
  • Offer informative feedback
    • all actions should result in system feedback
  • Design dialogues to yield closure. Don't provide meaningless or too technical messages.
  • Offer error prevention and simple error handling
  • Permit easy reversal of actions
  • Reduce short term memory load (7 +/- 2). Examples?

Feedback
  • when an action is initiated either the user should immediately see/hear an indication that the computer is responding.
  • if the user clicks a button and gets no feedback he/she is likely to click the button again (and again)
  • if the task is going to take longer than a second then the interface should display a progress bar
    • progress bar should show the relative amount of time remaining
    • when predictable: at each stage in a process you can update the current status of the menu bar: reading data..., processing images..., computing the results...
    • can also show a numeric countdown of time remaining if that is predictable (e.g. copying a file of known size)
